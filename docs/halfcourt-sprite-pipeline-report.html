Runtime Decision

The best runtime format for this iOS SpriteKit app is source PNG frames inside .atlas folders. Xcode compiles those folders into optimized .atlasc bundles at build time, and SpriteKit loads them with SKTextureAtlas.

Free Texture Packer is a valid open-source atlas packer, but it is a better fit when the engine needs a standalone spritesheet plus JSON or plist metadata. Here, SpriteKit already has native atlas compilation, so the simpler and more durable target is .atlas.

Authoring Tools

  • Krita: recommended open-source tool for clean hand-drawn frame animation.
  • Pixelorama: good open-source option for pixel-art sprites and sheets.
  • LibreSprite: open-source sprite editor descended from older Aseprite code.
  • Aseprite: strong sprite workflow, but not the open-source default here.

Runtime Wiring

  • HalfCourtSpriteFactory loads player and ball textures from SKTextureAtlas.
  • PlayerNode now prefers atlas animations and keeps the old procedural body as fallback.
  • BallNode now prefers animated spin textures and keeps the old shape-node ball as fallback.
  • HalfCourtHeroScene maps game state to idle, dribble, run, and shoot animations.

Verification

  1. Generation: npm run sprites:halfcourt produced 120 player frames and 12 ball frames.
  2. Manifest: JSON parsed successfully with frame size 256x256, four heroes, 120 player frames, and 12 ball frames.
  3. XcodeGen: xcodegen generate picked up the new Swift file and resource atlases.
  4. Build: xcodebuild -project BandMusicGames.xcodeproj -scheme BandMusicGames -destination 'generic/platform=iOS Simulator' build succeeded.
  5. Bundle: the built app contains HalfCourtHeroPlayers.atlasc and HalfCourtHeroBall.atlasc.
